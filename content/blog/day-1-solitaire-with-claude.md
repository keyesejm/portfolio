## One App a Day

I'm starting a challenge: **build one complete app per day using Claude Code**. The goal is to explore AI-assisted development, experiment with different tech stacks, and ship working products quickly.

### Day 1: Solitaire Game

**Live Demo:** [Play Solitaire →](https://solitaire-motu.vercel.app/)

I built a fully functional **Klondike Solitaire** game from scratch. It features:

- **Full game mechanics** — deck, tableau, foundations, and proper move validation
- **Drag-and-drop interface** — intuitive card manipulation
- **Game state management** — undo moves, restart game, track move count
- **Settings** — configure game rules (draw 1 or 3 cards, etc.)
- **Responsive design** — works on desktop and mobile
- **Instant deploy** — live on Vercel in minutes

### Tech Stack

- **React** — component-based UI
- **TypeScript** — type safety
- **Tailwind CSS** — styling
- **Vercel** — hosting and auto-deployment

### What I Learned

1. **Claude Code excels at game logic** — The card validation, move legality checking, and game state transitions were built quickly with AI assistance.

2. **Drag-and-drop is complex but doable** — Implementing smooth drag interactions took more iteration than expected, but the final result is polished.

3. **Deployment is the easy part** — Pushing to GitHub and deploying to Vercel happened in seconds. The real work was the game logic and UI.

4. **One day is tight but achievable** — Going from zero to a playable game in ~4-5 hours of focused work is possible with AI coding assistance.

### What's Next

- **Day 2**: Building another app (thinking a todo app with cloud sync)
- **Day 3+**: More variety — tools, games, utilities, experiments

The goal isn't to build production-grade apps (though they're deployed live). It's to **explore rapid prototyping, experiment freely, and see what's possible** when AI handles the coding grunt work.
